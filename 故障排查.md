# 故障排查

## 1. CPU 飙升
**现象**：`top` 看到某个进程 CPU 使用率接近 100%
**排查**：
- `top -o %CPU` 找到 PID
- `ps aux | grep PID` 确认进程
- `kill -9 PID` 终止异常进程
**预防**：生产环境避免死循环脚本，用 `crontab` 限制执行时间

## 2. 磁盘空间满
**现象**：写入文件报错 `No space left on device`
**排查**：
- `df -h` 看哪个分区满了
- `du -sh * | sort -rh | head -10` 找大文件
- `lsof | grep deleted` 找已删除但未释放的文件
**解决**：删除无用文件，或重启持有已删除文件的进程

## 3. 端口被占用
**现象**：启动服务报 `Address already in use`
**排查**：
- `ss -tlnp | grep 端口号` 看谁占用了
- 如果是容器：`docker ps` 找对应容器，`docker stop` 或 `docker rm -f`
- 如果是系统进程：`kill -9 PID`

## 4. Docker 容器起不来
**常见原因**：
- 端口冲突 → `ss -tlnp` 检查
- 名称冲突 → `docker ps -a` 找到旧容器，`docker rm -f`
- 镜像不存在 → `docker pull` 拉取
- 内存/CPU 不足 → `docker stats` 查看资源
- 挂载卷权限问题 → 检查宿主机目录权限

## 5. MySQL 连接数过多
**排查**：
- `SHOW PROCESSLIST;` 看当前连接
- `SHOW VARIABLES LIKE 'max_connections';` 看最大连接数
- 调整 `max_connections` 或优化慢查询
