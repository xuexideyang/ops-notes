# Zabbix

## 主动模式 vs 被动模式
- **被动模式**（默认）：Server 主动向 Agent 拉取数据。
- **主动模式**：Agent 主动向 Server 上报数据，适合远程网络不稳定或防火墙限制场景。

## 核心术语
- **Host**：被监控的主机。
- **Item**：监控项，如 CPU 使用率。
- **Trigger**：触发器，定义阈值条件。
- **Action**：动作，触发后的操作（如发告警）。
- **Template**：模板，预定义的一组监控配置，可批量应用。

## 模板
- 预先定义好的监控项、触发器、图形等配置的集合。
- 作用：批量部署相同类型的监控，方便统一维护和修改。

## 自定义监控项
- 在 `zabbix_agentd.conf` 中配置 `UserParameter=key,command`。
- 让 Zabbix 能够执行自定义脚本获取数据。

## 低级发现
- 自动发现动态资源（如磁盘分区、网卡、数据库实例）。
- 然后自动为每个资源创建监控项、触发器。

## 告警升级机制
- 防止告警无人处理，设置重发告警次数和间隔。
- 若问题未解决，自动升级到更高层级（如电话通知）。
