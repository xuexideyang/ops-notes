# 网络基础

## TCP 三次握手
- 目的：建立可靠连接，确认双方收发能力正常，同步序列号。
- 过程：
  1. 客户端发送 SYN（seq=x）— 表示想要连接。
  2. 服务端回复 SYN+ACK（seq=y, ack=x+1）— 表示同意并确认。
  3. 客户端回复 ACK（ack=y+1）— 连接建立成功。
- 为什么是三次不是两次：防止已失效的连接请求报文突然到达，避免浪费资源。

## TCP 四次挥手
- 目的：可靠地断开连接，确保数据完整传输。
- 过程：
  1. 客户端发送 FIN — 表示数据发送完毕，准备断开。
  2. 服务端回复 ACK — 表示确认收到。
  3. 服务端发送 FIN — 表示服务端数据也发送完毕。
  4. 客户端回复 ACK — 连接彻底断开。
- 为什么是四次不是三次：服务端收到 FIN 时可能还有数据要发，不能把 ACK 和 FIN 合并成一次，需要分两步走。

## HTTP 与 HTTPS 区别
- HTTP：明文传输，端口 80。
- HTTPS：加密传输（SSL/TLS），端口 443，需要证书。

## DNS 解析流程
